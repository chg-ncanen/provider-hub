# Scripts

Automation scripts and CLI tools for local use (not deployed).

## Structure

- `team/` — Shared scripts available to all team members (organized by team area: provider, pde, web)
- `user/` — Individual user scripts

## How to Add a Script

1. Create a directory under `team/<team-area>/` or `user/<your-name>/`
2. Include script files, dependencies list, and usage documentation
3. Submit a PR for review

## Available Scripts

Check subdirectories for individual script documentation.
